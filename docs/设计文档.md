## 一、项目定位

### 1.1 一句话描述

一个集**知识库、简历、项目管理、开源贡献、演示文稿**于一体的技术型个人门户，既是线上名片，也是数字资产沉淀中心。

### 1.2 目标用户

| 用户 | 诉求 |
|------|------|
| 面试官/HR | 快速了解技能、项目经历、技术深度 |
| 同行开发者 | 翻阅知识库文章、参考开源项目 |
| 合作方/社区 | 跟踪项目进展、获取演示材料 |
| 未来的自己 | 知识资产化、可搜索、可检索 |

### 1.3 核心价值主张

**「所见即所得的知识积累」**——阅读文章时看到的是网站，导出时是一份完整知识库（HTML+本地图片），随时随地可迁移到任何平台。

│  │ ...                                      │ │

**分类页 (/knowledge/{cat})：**
- 左侧分类导航树（展开当前分类）
- 右侧文章列表
- 每篇文章显示：标题、摘要（提取文章首200字）、标签、日期
- 支持按标签筛选

**文章阅读页 (/knowledge/{id})：**
- 文章正文（继承已有 HTML 样式）
- 右侧目录（TOC，从 h1/h2 自动生成）
- 底部：上一篇/下一篇导航
- 分享按钮
- 阅读进度条

**搜索功能：**
- 客户端全文搜索（Fuse.js 或 Pagefind）
- 搜索范围：标题 + 正文内容
- 高亮匹配结果

**技术实现：**
- 已有 52 篇 HTML 文章 → 构建时自动生成索引 JSON
- 文章 `article_body.json` → 提取纯文本用于搜索索引
- 分类信息从目录结构自动生成

## 四、技术架构

### 4.1 技术选型

| 层级 | 技术 | 原因 |

| 域名 | 待定 | 建议买个人域名（如 `name.dev`） |

## 五、视觉设计方向

### 5.1 设计原则

- **极简但不简陋**——内容为王，装饰服务于内容
- **暗色为主**——技术网站标配，减少眼睛疲劳
- **动画恰如其分**——Framer Motion 做入场/过渡动画，不过度
- **响应式优先**——Mobile First，所有页面适配手机
- **中文优化**——字体、排版针对中文阅读体验调优

## 六、知识库集成方案

### 6.1 现有资产

```
intern-learn/
├── agent-framework/      13篇 · 本地图已就绪
├── ai-explore/            4篇
├── business/              3篇
├── game-dev/             17篇
├── harness-engineering/    6篇
└── llm-wiki/              9篇（含三节子目录）
```

每篇文章包含：
- `index.html` — 完整文章 HTML（含本地图片引用）
- `images/` — 本地图片文件
- `article_body.json` — 文章正文 JSON

### 6.2 构建时处理流程

```
1. 扫描 knowledge/ 目录
2. 对每篇文章：
   a. 解析 index.html → 提取标题、摘要、关键词
   b. 提取所有 <h1>/<h2> → 生成 TOC
   c. 提取纯文本 → 加入搜索索引
3. 按分类汇总 → 生成 knowledge-index.json
4. 生成搜索倒排索引
```

### 6.3 文章阅读页

- 直接内嵌 `index.html` 内容（保留原有样式）
- 文章页面头部加入导航面包屑：首页 > 知识库 > 分类 > 文章名
- 右侧固定 TOC 栏，滚动高亮当前章节
- 底部：标签云 + 上一篇/下一篇
